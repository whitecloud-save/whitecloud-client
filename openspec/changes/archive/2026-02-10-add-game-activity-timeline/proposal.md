# 变更：添加游戏动态时间线展示功能

## 为什么

用户需要查看游戏相关的活动动态，包括游戏时长记录、存档备份状态等信息，以便更好地了解游戏使用情况和存档同步状态。

## 变更内容

- 新增游戏动态数据库实体，实现存档备份和上传失败的数据持久化
- 使用 ng-zorro 时间线组件展示游戏动态
- 时间标签使用 nzLabel 单独显示，格式为 YYYY-MM-DD hh:MM
- 支持展示四种类型的游戏动态：
  1. 游戏时长记录（在{主机名}上游玩了{游戏时间}），直接从 GameHistoryDB 读取
  2. 存档备份成功（未登录）
  3. 存档备份并上传成功（已登录）
  4. 存档上传失败（存档上传失败：{失败原因}）
- 为不同类型的动态提供视觉区分

## 影响

- 受影响规范：game-management
- 受影响代码：
  - 新增游戏动态数据库实体（GameActivity）
  - 在 game-home.component.html 的第 51 行集成时间线组件（已标记位置）
  - 扩展游戏服务以提供活动数据
  - 新增游戏动态仓库服务用于数据持久化操作
  - game-home.component.ts 需要注入游戏动态服务
