## 新增需求

### 需求：游戏动态数据持久化

系统必须提供游戏动态数据的持久化存储功能，确保存档备份和上传失败等活动记录能够永久保存并在需要时查询展示。

#### 场景：存储存档备份成功记录（未登录）

- **当** 用户未登录且存档备份成功
- **那么** 系统必须创建一条"存档备份成功（未登录）"类型的动态记录
- **并且** 记录必须包含：游戏ID、动态类型、创建时间
- **并且** 记录必须持久化存储到本地 SQLite 数据库

#### 场景：存储存档备份并上传成功记录（已登录）

- **当** 用户已登录且存档备份成功并上传至云存储
- **那么** 系统必须创建一条"存档备份并上传成功"类型的动态记录
- **并且** 记录必须包含：游戏ID、动态类型、创建时间
- **并且** 记录必须持久化存储到本地 SQLite 数据库

#### 场景：存储存档上传失败记录

- **当** 存档上传失败
- **那么** 系统必须创建一条存档上传失败类型的动态记录
- **并且** 记录必须包含：游戏ID、动态类型、失败原因、创建时间
- **并且** 记录必须持久化存储到本地 SQLite 数据库

#### 场景：查询游戏动态列表

- **当** 用户查看游戏首页（game-home.component）
- **那么** 系统必须查询并合并该游戏的所有动态记录：
  - 从 GameActivity 查询存档备份成功、存档备份并上传成功、存档上传失败记录
  - 从 GameHistoryDB 查询游戏时长记录
- **并且** 结果必须按时间倒序排列（合并后的时间）
- **并且** 查询结果必须在 game-home.component.html 第 51 行标记位置渲染时间线组件

#### 场景：数据库实体设计

- **当** 设计游戏动态数据库实体
- **那么** 实体必须包含以下字段：
  - id：主键，UUID
  - gameId：关联游戏ID
  - type：动态类型（枚举值），包含以下类型：
    - SAVE_BACKUP_LOCAL：存档备份成功（未登录）
    - SAVE_BACKUP_CLOUD：存档备份并上传成功（已登录）
    - SAVE_UPLOAD_FAILED：存档上传失败
  - content：动态内容（JSON 或文本）
  - createdAt：创建时间
  - updatedAt：更新时间
- **并且** 游戏时长记录不需要存储在 GameActivity 中，直接从 GameHistoryDB 读取
- **并且** 必须使用 TypeORM 进行 ORM 映射
- **并且** 必须配置 synchronize 模式自动同步数据库结构

### 需求：游戏动态时间线展示

系统必须提供游戏动态时间线功能，用于展示游戏相关的活动记录，包括游戏时长、存档备份状态等信息。

#### 场景：展示游戏时长记录

- **当** 用户在某个主机上游玩游戏
- **那么** 系统应在时间线上显示 "在{主机名}上游玩了{游戏时间}"
- **并且** 时间使用 nzLabel 标签单独显示
- **并且** 时间格式为 YYYY-MM-DD hh:MM
- **并且** 游戏时长数据必须从 GameHistoryDB 读取（基于 startTime 和 endTime 计算）

#### 场景：展示存档备份成功（未登录）

- **当** 用户未登录且存档备份成功
- **那么** 系统应在时间线上显示 "存档已备份"
- **并且** 时间使用 nzLabel 标签单独显示
- **并且** 时间格式为 YYYY-MM-DD hh:MM
- **并且** 使用视觉标识表示成功状态

#### 场景：展示存档备份并上传成功（已登录）

- **当** 用户已登录且存档备份成功并上传至云存储
- **那么** 系统应在时间线上显示 "存档已备份（并上传至云储存）"
- **并且** 时间使用 nzLabel 标签单独显示
- **并且** 时间格式为 YYYY-MM-DD hh:MM
- **并且** 使用视觉标识表示成功状态

#### 场景：展示存档上传失败

- **当** 存档上传失败
- **那么** 系统应在时间线上显示 "存档上传失败：{失败原因}"
- **并且** 时间使用 nzLabel 标签单独显示
- **并且** 时间格式为 YYYY-MM-DD hh:MM
- **并且** 使用视觉标识表示失败状态
- **并且** 失败原因应准确反映实际错误信息

#### 场景：使用 ng-zorro 时间线组件

- **当** 渲染游戏动态时间线
- **那么** 必须使用 ng-zorro 的时间线组件（nz-timeline）
- **并且** 必须使用 nzLabel 标签单独显示时间
- **并且** 时间内容与事件内容分离显示
